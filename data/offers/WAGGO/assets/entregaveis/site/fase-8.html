Observação E1: Testimonials Placeholder

Severidade: Alta

Emails de social proof (Weekly #4, Post-Conv) usam testimonials placeholder. DEVEM ser substituídos por depoimentos reais antes do envio.

Não enviar com testimonials fabricados — isso viola FTC e destrói credibilidade.

Observação E2: Pro Upgrade Sequence

Severidade: Baixa

O Settle mencionou que não escreveu a Pro Upgrade sequence completa (3 emails). Apenas Post-Conv 3 serve como teaser. A sequence completa deve ser escrita após dados reais de uso.

Não é necessário no launch. Implementar após Month 1.

Observação E3: [Max] Personalization Token

Severidade: Média

Todos os emails usam [Max] como personalização. Precisa verificar com equipe de dev se o email system suporta merge tags dinâmicos (nome do pet + profile name).

Sem personalization, usar fallback genérico ("your pet").

Observação E4: Repulsion Email (Onboarding Day 5)

Severidade: Informativa

O email "WAGGO is not for everyone" é um soft repulsion. Para pet parents, o tom está correto (não tão agressivo quanto o Settle natural). Manter como está — funciona como filtro sem alienar.

Nenhuma ação necessária.
