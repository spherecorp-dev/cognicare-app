# Winners Library — squad-copy

> Biblioteca de criativos winners. Alimentada pelo @winners-analyst e pelo copy humano.
> Cada winner deve ter autopsia completa para retroalimentar o pipeline.

---

## Como usar este arquivo

- @winners-analyst adiciona autopsias automaticamente via *analyze-winners
- Copy humano pode adicionar manualmente (5 min/dia recomendado)
- @stefan-georgi consulta para gerar variacoes via *variate-winner
- @creative-deconstructionist consulta para comparar patterns

## Template de Winner

```
### Winner: [ID ou nome descritivo]

| Campo | Valor |
|-------|-------|
| **Oferta** | Nome da oferta |
| **Geo** | fr / es / en |
| **Formato** | imagem / carrossel / VSL / UGC / podcast / AI avatar |
| **Hook type** | curiosity / fear / desire / controversy / social_proof / pattern_interrupt |
| **Angulo** | Descricao do angulo usado |
| **Mecanismo** | Tipo de mecanismo (scientific / story / analogy / demonstration) |
| **Metricas** | ROAS: X | CPA: $X | CTR: X% | Spend: $X | Conversions: X |
| **Duracao ativo** | X dias |
| **Por que funcionou** | 1-2 frases explicando |
| **Potencial de replicacao** | high / medium / low |
| **Variacoes sugeridas** | hook_swap, tone_shift, etc |
| **Data** | YYYY-MM-DD |
```
