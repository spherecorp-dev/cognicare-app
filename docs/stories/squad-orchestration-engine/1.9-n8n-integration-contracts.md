## Story

**As a** n8n workflow,
**I want** clear webhook contracts to trigger the squad engine and receive callbacks on completion,
**so that** I can orchestrate offers end-to-end (Notion → Squad Engine → Google Drive) automatically.

## Acceptance Criteria

1. **AC1 — Webhook Endpoint:** `POST /api/webhooks/n8n` implemented in `dashboard/src/app/api/webhooks/n8n/route.ts`. Accepts payload: `{ event: string, payload: { offerId: string, platforms?: string[], callback_url?: string, overrides?: object } }`.

2. **AC2 — API Key Authentication:** Validates API key in `Authorization: Bearer {token}` header. Key checked against `N8N_WEBHOOK_API_KEY` environment variable. Returns `401 Unauthorized` with `{ error: "Unauthorized" }` if invalid or missing.

3. **AC3 — Event Handling:** When `event === "offer.created"`, triggers squad-copy run via SquadOrchestrator with trigger data. Returns `{ success: true, runId: string }` immediately (async execution). Unknown events return `400 Bad Request` with `{ error: "Unknown event type" }`.

4. **AC4 — Rate Limiting:** Maximum 100 requests per minute per API key. Implemented using in-memory counter (Map with TTL). Returns `429 Too Many Requests` with `{ error: "Rate limit exceeded", retryAfter: seconds }` when exceeded.

5. **AC5 — Callback Registration:** If `payload.callback_url` is provided, stores URL in run state (`trigger.callback_url`). Callback is sent when run completes (or fails/aborts).

6. **AC6 — Callback Execution:** On run completion, sends `POST {callback_url}` with body: `{ runId, status, duration_ms, outputs_path, error?: string }`. Content-Type: application/json.

7. **AC7 — Callback Retry:** If callback POST fails (connection error, timeout, 5xx), retry 3 times with exponential backoff (2s, 4s, 8s). After 3 failures, log warning and stop retrying. Callback failures do NOT affect run status.

8. **AC8 — Webhook Logging:** All webhook requests logged to `.aios/webhooks/log.jsonl` in append-only format: `{ timestamp, event, offerId, apiKeyHash (last 4 chars), runId, status, ip }`. Callback results also logged.

9. **AC9 — Input Validation:** Validate webhook payload:
   - `event` is required string
   - `payload.offerId` is required string
   - `payload.platforms` is optional array of strings
   - `payload.callback_url` is optional valid URL
   - `payload.overrides` is optional object
   - Invalid payload returns `400 Bad Request` with `{ error: "Validation failed", details: [...] }`

## Dev Notes

### Risks

- **SSRF via callback_url:** Malicious actor could register internal URLs (localhost, 10.x.x.x) como callback, forçando o server a fazer requests internos. Mitigação: validar callback_url contra lista de IPs privados (Task 4 subtask adicionado).
- **API key exposure:** Se N8N_WEBHOOK_API_KEY vazar, qualquer um pode triggerar runs. Mitigação: rate limiting (AC4) limita blast radius. Key rotation é OUT of scope para MVP.
- **Callback reliability:** Se n8n endpoint estiver down, callbacks falham silenciosamente após 3 retries. Mitigação: webhook log permite auditoria manual.

### Previous Stories Insights

- **API Route Pattern:** Next.js 16 route handlers. Use `export async function POST(req: Request)`. [Source: Story 1.5]
- **File System Logging:** Follow same append-only JSONL pattern as EventStore (Story 1.7). [Source: PRD]
- **Environment Variables:** Add to `.env.example` but never commit real keys. [Best practice]
- **Error Handling:** Use `catch (error: unknown)` with `(error as Error).message`. [Source: Story 1.5]

### Rate Limiter Design

```typescript
// In-memory sliding window counter
const rateLimits = new Map<string, { count: number; resetAt: number }>();

function checkRateLimit(apiKeyHash: string): { allowed: boolean; retryAfter?: number } {
  const now = Date.now();
  const window = 60_000; // 1 minute
  const limit = 100;

  const entry = rateLimits.get(apiKeyHash);
  if (!entry || entry.resetAt <= now) {
    rateLimits.set(apiKeyHash, { count: 1, resetAt: now + window });
    return { allowed: true };
  }

  if (entry.count >= limit) {
    return { allowed: false, retryAfter: Math.ceil((entry.resetAt - now) / 1000) };
  }

  entry.count++;
  return { allowed: true };
}
```
