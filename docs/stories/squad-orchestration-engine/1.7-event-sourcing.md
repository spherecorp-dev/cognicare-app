## Story

**As a** AIOS administrator,
**I want** a complete event history for squad executions in an append-only log,
**so that** I can audit, debug, and reconstruct the state of any run — even if state.yaml is corrupted.

---

## Acceptance Criteria

1. **AC1 — EventStore Class:** `EventStore` implemented in `.aios-core/core/orchestration/squad-engine/event-store.js` with methods: `append(runId, event)`, `replay(runId)`, `getEvents(runId, filter?)`.

2. **AC2 — Append-Only Log:** All events saved to `.aios/squad-runs/{runId}/events.jsonl` using append-only writes (no file rewrite). Each line is a valid JSON object.

3. **AC3 — Event Types:** The following events are emitted by the orchestration engine:
   - `run.started` — When a new run begins (data: squadId, trigger, playbook)
   - `phase.started` — When a phase begins execution (data: phaseName, phaseIndex)
   - `phase.completed` — When a phase finishes (data: phaseName, status, duration_ms)
   - `step.started` — When a step begins (data: stepId, stepType, input)
   - `step.completed` — When a step finishes (data: stepId, status, output, duration_ms)
   - `gate.evaluated` — When a gate is evaluated (data: gateId, verdict, round)
   - `run.paused` — When run is paused (data: reason, current_step)
   - `run.resumed` — When run is resumed (data: resumed_at_step)
   - `run.completed` — When run finishes (data: status, total_duration_ms, outputs_count)
   - `run.failed` — When run fails (data: error, failed_step, stack)
   - `run.aborted` — When run is aborted (data: reason, aborted_at_step)

4. **AC4 — Event Schema:** Each event has the structure: `{ event: string, timestamp: string (ISO 8601), runId: string, data: object }`. Timestamp is auto-generated at append time.

5. **AC5 — Replay Capability:** `EventStore.replay(runId)` reads all events from events.jsonl and reconstructs the complete run state (status, currentPhase, currentStep, completedPhases, context). Returns reconstructed state object compatible with StateManager format.

6. **AC6 — State Recovery:** If `state.yaml` is missing or corrupted (invalid YAML/checksum mismatch), `StateManager` falls back to `EventStore.replay(runId)` to reconstruct state. Recovery logged as `state.recovered` event.

7. **AC7 — Dashboard Timeline API:** `GET /api/runs/{runId}/events` endpoint returns paginated events (latest 50, with `?after=timestamp` cursor). Response format: `{ events: [...], hasMore: boolean, nextCursor: string }`.

8. **AC8 — Integration with Orchestrator:** `SquadOrchestrator`, `TaskExecutor`, and `StateManager` emit events at appropriate lifecycle points. Events are fire-and-forget (non-blocking) — event write failure does NOT stop pipeline execution.

9. **AC9 — Performance:** Appending an event takes <5ms. Replaying 1000 events takes <500ms. File I/O uses `fs.appendFileSync` for atomicity.

### JSONL Format Reference

```jsonl
{"event":"run.started","timestamp":"2026-02-21T10:00:00.000Z","runId":"run_abc123","data":{"squadId":"squad-copy","trigger":{"type":"manual"},"playbook":"pipeline.yaml"}}
{"event":"phase.started","timestamp":"2026-02-21T10:00:01.000Z","runId":"run_abc123","data":{"phaseName":"intelligence","phaseIndex":0}}
{"event":"step.started","timestamp":"2026-02-21T10:00:02.000Z","runId":"run_abc123","data":{"stepId":"fetch-offer-data","stepType":"task_pura","input":{"offerId":"offer_456"}}}
{"event":"step.completed","timestamp":"2026-02-21T10:00:05.000Z","runId":"run_abc123","data":{"stepId":"fetch-offer-data","status":"completed","output":{"offer_context":{}},"duration_ms":3000}}
```

### Risks

- **Replay com muitos eventos:** Para runs longos (>10K eventos), replay pode exceder 500ms. Mitigação: pagination no replay ou snapshot periódico (future).
- **Concurrent writes:** Múltiplos componentes escrevendo em events.jsonl simultaneamente pode causar corrupção. Mitigação: `fs.appendFileSync` é atômico para writes pequenos (<4KB no Linux). Para MVP é suficiente.
- **Disk space:** events.jsonl cresce indefinidamente. Mitigação: OUT of scope (rotation/compression é future).

### Performance Requirements

- `append()`: <5ms (single fs.appendFileSync call)
- `replay(1000 events)`: <500ms
- `getEvents(50 limit)`: <100ms
