## User Story

**As a** Squad Orchestration Engine,
**I want** validar pre-conditions ANTES e post-conditions DEPOIS de cada task,
**so that** posso garantir integridade de execução e bloquear steps inválidos.

---

## Description

Este story implementa o ConditionEngine, componente responsável por validar conditions antes e depois da execução de tasks. O ConditionEngine deve:

1. Validar pre-conditions ANTES de executar uma task
2. Validar post-conditions DEPOIS de executar uma task
3. Bloquear execução se condition com `blocker=true` falhar
4. Resolver `source` no context acumulado (ex: `{{fetch-offer-data.output.offer_context}}`)
5. Registrar failures em event log
6. Retornar resultado estruturado com failures detalhadas

**Contexto Técnico:**

O ConditionEngine garante integridade da execução do pipeline ao validar que:
- Dados necessários existem antes de uma task executar (pre-conditions)
- Output de uma task atende critérios mínimos (post-conditions)
- Pipeline pode parar gracefully se blocker condition falhar

**Integração com Sistema Existente:**

- Integra com `TaskExecutor` (Story 1.2) para validação antes/depois de cada task
- Reutiliza lógica de interpolação de variáveis para resolver `source` paths
- Novo componente: `.aios-core/core/orchestration/squad-engine/condition-engine.js`

## Integration Verification

### IV1: Pre-condition "Offer data fetched" bloqueia se fetch-offer-data não rodou

**Test:** Pre-condition blocker impede execução de task

```javascript
const engine = new ConditionEngine();
const conditions = [
  {
    condition: 'Offer data fetched',
    source: '{{fetch-offer-data.output.offer_context}}',
    validation: 'offer_context presente',
    blocker: true
  }
];

const context = {}; // fetch-offer-data não rodou ainda

const result = await engine.validate(conditions, context);

assert(result.all === false);
assert(result.blockersFailed === true);
assert(result.failures.length === 1);
assert(result.failures[0].condition === 'Offer data fetched');
```

### Condition Schema

```yaml
condition:
  condition: string           # Human-readable description
  source: string             # Context path to resolve (ex: "{{step.output.field}}")
  validation: string         # Validation expression (ex: "field presente", "count >= 3")
  blocker: boolean           # If true, failure stops execution
  error_message: string?     # Optional custom error message
```

### Validation Syntax

**Supported validations:**
- `{field} presente` — Field exists and is not null/undefined
- `{field} >= {value}` — Numeric comparison
- `{field} <= {value}` — Numeric comparison
- `{field} == {value}` — Equality check
- `{field}.length >= {value}` — Array length check

**Future enhancements (NOT in this story):**
- Regex matching
- Custom validation functions
- Complex boolean logic (AND/OR)

### Error Handling

**PreConditionError:**
```javascript
class PreConditionError extends Error {
  constructor(failures) {
    super(`Pre-conditions failed: ${failures.map(f => f.condition).join(', ')}`);
    this.name = 'PreConditionError';
    this.failures = failures;
  }
}
```

**PostConditionError:**
```javascript
class PostConditionError extends Error {
  constructor(failures) {
    super(`Post-conditions failed: ${failures.map(f => f.condition).join(', ')}`);
    this.name = 'PostConditionError';
    this.failures = failures;
  }
}
```

### Integration with TaskExecutor

**Modified TaskExecutor.executeTask():**
```javascript
async executeTask(step, context, squadId) {
  // 1. Validate pre-conditions
  if (step.pre_conditions) {
    const result = await this.conditionEngine.validate(
      step.pre_conditions,
      context
    );
    if (result.blockersFailed) {
      throw new PreConditionError(result.failures.filter(f => f.blocker));
    }
  }

  // 2. Execute task (existing logic)
  const output = await this.executePureTask(step, context, squadId)
    || await this.executeAgentTask(step, context, squadId);

  // 3. Validate post-conditions
  if (step.post_conditions) {
    const result = await this.conditionEngine.validate(
      step.post_conditions,
      { ...context, [step.id]: { output } }
    );
    if (result.blockersFailed) {
      throw new PostConditionError(result.failures.filter(f => f.blocker));
    }
  }

  return output;
}
```
