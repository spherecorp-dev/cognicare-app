**I want** endpoints REST para criar, listar e controlar runs de squad,
**so that** posso exibir informações em tempo real e permitir controle de execuções.

---

## Description

Este story implementa os endpoints de API REST necessários para o dashboard do Squad Orchestration Engine. A API fornece acesso programático para:

1. **Criar execuções** de squads com triggers customizados
2. **Listar execuções** ativas e históricas (paginado)
3. **Consultar estado** de execução em tempo real (para polling)
4. **Controlar execuções** (pause, resume, abort)
5. **Acessar logs** e outputs gerados

**Contexto Técnico:**

Os endpoints usam Next.js API Routes (App Router) e integram-se com os componentes do Squad Engine implementados nas stories 1.1-1.4:
- `SquadOrchestrator` - para criar e gerenciar runs
- `StateManager` - para consultar e manipular estado
- `TaskExecutor` - para acessar contexto e outputs

**Integração com Sistema Existente:**

- Segue padrão Next.js existente: `dashboard/src/app/api/runs/`
- Reutiliza estrutura de armazenamento: `.aios/squad-runs/{runId}/`
- Integra com SquadOrchestrator (Story 1.1)
- Usa StateManager (Story 1.4) para operações de estado
- Novo componente: API routes em `dashboard/src/app/api/runs/`

### API Contracts

**POST /api/runs**
```typescript
// Request
{
  squadId: string;           // Required: 'squad-copy', 'squad-trafego', etc.
  trigger: {                 // Required: trigger context
    [key: string]: any;      // Squad-specific trigger data
  };
}

// Response (201 Created)
{
  runId: string;             // Generated run identifier
  status: 'running';         // Initial status
  squadId: string;           // Echo back squadId
  startedAt: string;         // ISO 8601 timestamp
}

// Error Response (400 Bad Request)
{
  error: string;             // Human-readable error message
  details: {                 // Validation details
    field: string;
    message: string;
  }[];
}
```

**GET /api/runs**
```typescript
// Query Parameters
?limit=50                    // Optional: items per page (default: 50, max: 100)
&offset=0                    // Optional: pagination offset (default: 0)

{
  runs: Array<{
    runId: string;
```

### Error Handling Strategy

**HTTP Status Codes:**
- `200 OK` - Successful GET request
- `201 Created` - Successful POST /api/runs
- `400 Bad Request` - Invalid payload or parameters
- `404 Not Found` - Run ID not found
- `409 Conflict` - Invalid state transition (e.g., resume when not paused)
- `500 Internal Server Error` - Unexpected server error

**Error Response Format:**
```javascript
{
  error: string,              // Human-readable message
  code: string,               // Machine-readable error code (e.g., 'RUN_NOT_FOUND')
  details?: object,           // Additional context (optional)
  timestamp: string           // ISO 8601 timestamp
}
```

**Implementation Pattern:**
```javascript
// dashboard/src/app/api/runs/[runId]/state/route.ts
export async function GET(request, { params }) {
  try {
    const { runId } = params;

    // Validate runId format
    if (!runId || typeof runId !== 'string') {
      return NextResponse.json(
        { error: 'Invalid run ID', code: 'INVALID_RUN_ID' },
        { status: 400 }
      );
    }

    // Load state
    const stateManager = new StateManager(runId);
    const state = await stateManager.loadState();

    if (!state) {
      return NextResponse.json(
        { error: 'Run not found', code: 'RUN_NOT_FOUND', runId },
        { status: 404 }
      );
    }

    return NextResponse.json(state, { status: 200 });

  } catch (error) {
    console.error('Error loading run state:', error);
    return NextResponse.json(
      {
        error: 'Internal server error',
        code: 'INTERNAL_ERROR',
        message: error.message
      },
      { status: 500 }
    );
  }
}
```

### Performance Considerations

**Optimization Strategies:**

1. **File System Caching:**
```javascript
// Cache run list in memory for 5 seconds
let runListCache = { data: null, timestamp: 0 };
const CACHE_TTL = 5000;

async function listRuns() {
  const now = Date.now();
  if (runListCache.data && (now - runListCache.timestamp) < CACHE_TTL) {
    return runListCache.data;
  }

  const runs = await scanRunsDirectory();
  runListCache = { data: runs, timestamp: now };
  return runs;
}
```

2. **Pagination:**
```javascript
// Only read metadata files, not full state.yaml
async function listRuns(limit = 50, offset = 0) {
  const runDirs = await fs.readdir('.aios/squad-runs/');
  const total = runDirs.length;

  const paginatedDirs = runDirs
    .slice(offset, offset + limit);

  const runs = await Promise.all(
    paginatedDirs.map(async (runId) => {
      // Read only minimal metadata, not full state
      return await loadRunMetadata(runId);
    })
  );

  return {
    runs,
    pagination: {
      total,
      limit,
      offset,
      hasNext: offset + limit < total
    }
  };
}
```

3. **Log Streaming:**
```javascript
// Read only last N lines instead of full file
async function getLastNLines(filePath, n = 100) {
  const stats = await fs.stat(filePath);
  const chunkSize = Math.min(stats.size, n * 200); // Estimate 200 bytes/line

  const buffer = Buffer.alloc(chunkSize);
  const fd = await fs.open(filePath, 'r');
  await fd.read(buffer, 0, chunkSize, stats.size - chunkSize);
  await fd.close();

  const lines = buffer.toString().split('\n').slice(-n);
  return lines;
}
```

### Integration with SquadOrchestrator

**Creating Runs:**
```javascript
// POST /api/runs implementation
import { SquadOrchestrator } from '@/aios-core/core/orchestration/squad-engine';

export async function POST(request) {
  const { squadId, trigger } = await request.json();

  const orchestrator = new SquadOrchestrator();
  const runId = await orchestrator.createRun(squadId, trigger);

  // Start execution (non-blocking)
  orchestrator.executeRun(runId).catch(error => {
    console.error(`Run ${runId} failed:`, error);
  });

  return NextResponse.json({
    runId,
    status: 'running',
    squadId,
    startedAt: new Date().toISOString()
  }, { status: 201 });
}
```

**Controlling Runs:**
```javascript
// POST /api/runs/[runId]/control implementation
import { StateManager } from '@/aios-core/core/orchestration/squad-engine';

export async function POST(request, { params }) {
  const { runId } = params;
  const { action } = await request.json();

  const stateManager = new StateManager(runId);

  switch (action) {
    case 'pause':
      await stateManager.pause();
      return NextResponse.json({
        runId,
        action,
        status: 'paused',
        message: 'Run paused successfully'
      });

    case 'resume':
      await stateManager.resume();
      return NextResponse.json({
        runId,
        action,
        status: 'running',
        message: 'Run resumed successfully'
      });

    case 'abort':
      await stateManager.abort();
      return NextResponse.json({
        runId,
        action,
        status: 'aborted',
        message: 'Run aborted successfully'
      });

    default:
      return NextResponse.json({
        error: 'Invalid action',
        validActions: ['pause', 'resume', 'abort']
      }, { status: 400 });
  }
}
```
