## Story

**As a** Squad Orchestration Engine,
**I want** cachear resultados de tasks com idempotency keys,
**so that** retries não executem tasks 2x (ex: gerar mesma imagem 2x na API).

---

## Acceptance Criteria

1. **AC1 — Idempotency Key Generation:** `TaskExecutor` gera hash criptográfico de `(runId, step.id, JSON.stringify(input))` como idempotency key usando SHA256.

2. **AC2 — Cache Check Before Execution:** Antes de executar qualquer task, `TaskExecutor` checa cache: `resultCache.get(key)`. Se cache hit, retorna resultado cacheado sem executar a task.

3. **AC3 — Cache Store After Execution:** Se cache miss, executa task normalmente, salva resultado em cache com `resultCache.set(key, result, ttl)`. TTL padrão: 24 horas.

4. **AC4 — LRU Cache Implementation:** Cache implementado como LRU (Least Recently Used) com max 1000 entries em memória (MVP). Classe `IdempotencyCache` em `.aios-core/core/orchestration/squad-engine/idempotency-cache.js`.

5. **AC5 — Cache Persistence Across Pause/Resume:** Cache state é serializado no `state.yaml` durante pause e restaurado durante resume. Cache entries com TTL expirado são descartadas no restore.

6. **AC6 — Logging:** Log quando task é skipped por cache hit: `step.cache_hit` event emitido via EventStore com data `{ stepId, idempotencyKey, cached_at }`.

7. **AC7 — Cache Bypass:** Suportar flag `force_execute: true` no step config para forçar execução mesmo com cache hit (bypass idempotency).

### LRU Cache Design

```javascript
class IdempotencyCache {
  constructor(maxEntries = 1000, defaultTTL = 86400000) { // 24h in ms
    this.cache = new Map(); // Map preserves insertion order
    this.maxEntries = maxEntries;
    this.defaultTTL = defaultTTL;
  }

  get(key) {
    const entry = this.cache.get(key);
    if (!entry) return null;
    if (Date.now() > entry.expiresAt) {
      this.cache.delete(key);
      return null;
    }
    // Move to end (most recently used)
    this.cache.delete(key);
    this.cache.set(key, entry);
    return entry.result;
  }

  set(key, result, ttl = this.defaultTTL) {
    // Evict LRU if at capacity
    if (this.cache.size >= this.maxEntries && !this.cache.has(key)) {
      const oldest = this.cache.keys().next().value;
      this.cache.delete(oldest);
    }
    this.cache.set(key, {
      result,
      cachedAt: Date.now(),
      expiresAt: Date.now() + ttl
    });
  }
}
```

### Risks

- **Memory pressure:** 1000 entries com outputs grandes podem consumir memória. Mitigação: LRU eviction + outputs são tipicamente pequenos (JSON objects, não binary).
- **State.yaml bloat:** Serializar cache no state.yaml pode aumentar tamanho significativamente. Mitigação: Apenas serializar keys + metadata, não full results. Ou usar arquivo separado `cache.json`.
- **Cache invalidation:** Se input de task muda levemente (timestamp diferente), cache miss. Mitigação: Input normalization antes do hash.
